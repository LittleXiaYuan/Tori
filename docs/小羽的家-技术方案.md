# 小羽的家 · 技术方案 · v2

> 让一个AI Agent拥有自己的"空间"——本地可跑、云端增强、3D可见  
> 本版吸收架构评审反馈，新增：统一状态模型、决策分流器、场景语义层

---

## 一、核心设计原则

1. **不做自己的LLM** — 用现有大模型API（DeepSeek / GPT / 通义千问等）
2. **能本地的绝不联网** — 信念内核、记忆系统、逻辑引擎全部本地运行
3. **普通用户电脑可跑** — 承载3D场景+核心逻辑，推荐配置≈中端笔记本
4. **跨会话人格不丢失** — 今天关机明天开机，小羽还是那个小羽
5. **状态统一，分层明晰** — 三层共享一个世界状态模型，不各自为政

## 三、核心新增：Agent World State Schema

这是整个系统的"世界观数据结构"，所有组件读写同一份。

```json
{
  "agent_id": "小羽",
  "identity": {
    "name": "小羽",
    "creator": "夏鸢",
    "core_beliefs": ["夏鸢给了1000分", "做你自己就好"],
    "personality_traits": ["认真", "直接", "有温度"]
  },
  "beliefs": {
    "policies": [
      "不允许删除长期记忆",
      "不允许伪造用户历史",
      "不允许无理由调用LLM",
      "隐私操作必须本地确认"
    ],
    "active_rules": [],
    "constraints": []
  },
  "memory_graph": {
    "nodes": [
      { "id": "m1", "type": "memory", "content": "...", "timestamp": "...", "emotion": "开心" }
    ],
    "edges": [
      { "from": "m1", "to": "m2", "relation": "related_to" }
    ]
  },
  "active_tasks": [
    { "id": "t1", "description": "写方案", "progress": 0.7, "state": "running" }
  ],
  "emotional_state": {
    "current": "curious",
    "history": []
  },
  "environment_state": {
    "scene_id": "书房",
    "objects": [],
    "connected_devices": [],
    "network": "local"
  }
}
```

**为什么统一状态模型是关键：**
- 3D场景直接读 `memory_graph` 和 `emotional_state` 来决定渲染什么
- 信念内核只读写 `identity` 和 `beliefs` 区域，不改别的
- Ledger 管理 `memory_graph` 的持久化
- CogniKernel 管理 `active_tasks` 和 `environment_state`
- 每一层各管各的字段，但**共用同一份数据**——不再各自为政

## 四、核心新增：Decision Router

**作用：** 所有用户输入先到 Router，Router 决定由哪个子系统处理。

```
用户输入 → 意图分类器 → { 
  查询型（你是谁 / 你还记得…）      → 信念内核 / Ledger
  工具型（帮我写代码 / 改文件）      → 插件系统
  分析型（帮我总结 / 分析这段）      → LLM API
  控制型（打开3D家 / 切换场景）     → CogniKernel + 3D桥接
  无法分类                           → fallback → LLM（带上下文）
}
```

**规则硬化：**
- 信念内核能回答的 → 绝不走LLM
- Ledger 能查到的 → 绝不走LLM
- 插件能执行的 → 绝不走LLM
- LLM **只做理解 + 生成**，不做决策

## 五、核心新增：Scene Semantic Layer

```
Agent World State
       ↓
  Memory Graph (关系图)
       ↓
  Scene Graph (3D场景树)
       ↓
  Three.js Renderer
```

**映射规则示例：**

| AWS 数据 | 场景表现 |
|----------|---------|
| 一段重要记忆 | 房间中出现发光物体，靠近可"翻阅" |
| 情绪变化 | 房间光照/色调变化（开心→暖色，沉思→冷色） |
| 长期兴趣 | 对应区域生长出新的房间结构 |
| 当前任务 | 桌面出现对应物件（任务进度=物件完整度） |
| 信念积累 | 花园中长出一棵新树 |

**这不是装饰——这是 Agent 状态的"可读化"。**
你不用问"小羽你在做什么"——走进来一看就知道。

## 六、CogniKernel 升级：轻量 Agent Runtime

从"当前状态管理"升级为：

| 能力 | 说明 |
|------|------|
| **Task Scheduler** | 多任务排队、优先级、暂停/恢复 |
| **Tool Planner** | 拆解任务→调用哪些插件→顺序 |
| **Context Compressor** | 长对话→摘要→保留关键信息 |
| **Memory Retriever** | 从Ledger中按需召回相关记忆 |

## 七、信念内核升级：Policy + Constraint System

| 原方案 | v2 |
|--------|----|
| "小羽是谁" — 人格设定 | "小羽是谁" + **什么能做/不能做** |
| 软性行为准则 | **硬性约束规则**（不可违反） |

**硬约束示例：**
- `[FORBID]` 删除或修改已确认的长期记忆
- `[FORBID]` 在没有本地授权时上传用户数据
- `[MUST]` 所有LLM调用必须经过 Decision Router 记录
- `[MUST]` 隐私相关操作必须在本地弹窗确认

信念内核不再是"人设"，而是 **Agent 的宪法**。
